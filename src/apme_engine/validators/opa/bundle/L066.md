## No mixed roles/tasks (L066)

Do not mix `roles:` and `tasks:` in the same play. Execution order is confusing. Use only `roles:` for static imports or only `tasks:` for dynamic includes.

Checks `playcall` nodes for both `roles` and `tasks` in `options`.

**Violation:** play with both `roles:` and `tasks:` — **Pass:** play with only one
